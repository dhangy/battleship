The game grid is always generated anew and dynamically using the createBoard Function. As you can see it is nicely styled using css. When you load the grid it has a checkerboard pattern of blues. Each cell color is updated according to what state it represents white indicated a miss gray indicates a ship and red indicates a hit on a ship

This game gets user input through the grid primarily but it also uses buttons to get the info the place the ships. It previously had more types of selection but i found them to be unnecessary and less visually pleasing.

There is dynamic modification of the HTML elements using the inner HTML each time a grid cell gets updated info on the model it gets updaetd on the view dynamically

it incorporates an audio tag on the game description page to satisfy the first HTML 5 requirement(as well as after hits on the grid page) and it uses local storage to store the wins record to satifsy the extra credit requirement

The game logic is obviously powering the game! Give the game a try to test that out!
